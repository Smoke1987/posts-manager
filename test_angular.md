# Описание
Нужно создать приложение по нижеприведенному заданию<br>
Продумать архитектуру приложения<br>
Создать необходимые компоненты и представления<br>

## Задание
### Источник данных
В качестве источника использовать демо-ответ на GET-запрос http://jsonplaceholder.typicode.com/posts<br>
Сохранить полученные данные в локальной переменной<br>
Обогатить данные расчетным полем **count** подсчитывающим количество постов в группировке по **userId**

### Интерфейс

#### Логин
* Сделать модальную форму логина для ролей пользователя (*user/user*) и администратора (*admin/admin*) с фиксированными в конфиге приложения логином/паролем
* При вводе неверных реквизитов выводить алерт с текстом ***Неверные реквизиты входа***
* Предусмотреть скрытие вводимого пароля
* Предусмотреть отображение зарегистрировавшегося логина в шапке
* Предусмотреть разлогин с запросом при клике на имя логина в шапке
#### Список
* Выводить список из полученных данных в виде таблицы
* Предусмотреть пагинацию и сортировку 
* Для роли **user** выводить посты с четными **id**, поля: *title*
* Для роли **admin** выводить все посты, поля: *userId, id, title, count*
* Для роли **admin** предусмотреть вывод поля **count** в подвале для выбранного **userId** активного поста. Для роли **user** - данная функциональность должна быть недоступна как элемент в DOM. 
* Клик ЛКМ или нажатие Enter на активной строке таблицы должно вести в представление **Детализация**
* Предусмотреть кнопку **Обновить** для повторного запроса данных

#### Детализация
* Выводить список из полученных данных с фильтрацией по **userId**, выбранный в активной строке представления **Список** в виде таблицы
* Предусмотреть пагинацию и сортировку 
* Для роли **user** выводить поля: *title, body*
* Для роли **admin**: *userId (выводить в заголовке), id, title, body*
* Предусмотреть кнопки **Добавить**, **Править**, **Удалить**, **Назад**. 
    * При нажатии на кнопки **Добавить** или **Править** должен происходить переход на представление **Правка**. В случае **Добавить** с пустыми полями для ввода, в случае **Править** с полями, заполенными данными текущего поста из активной строки представления **Детализация**
    * При нажатии на кнопку **Удалить** для роли **user** должен выводится алерт с текстом: ***Вам не доступно данное действие***
    * При нажатии на кнопку **Удалить** для роли **admin** должен выводится диалог с вопросом: ***Подтверждаете данное действие?*** и в случае подтверждения пост с данным **id** должен удаляться из сохраненных данных
    * При нажатии на кнопку **Назад** должен происходить возврат на представление **Список** с фиксацией на строке с выбранным **id** в представлении **Детализация** поста и происходить пересчет поля **count** если необходимо

#### Правка
* Выводить информацию в виде формы для текущего поста или создавать пустой пост. Снизу - кнопки **ОК** (неактивна) и **Отменить**
* Отображение, правка и валидация полей должны производиться в соответствии с типом. Предусмотреть правила заполнения полей в соответствии с ролями
* Для поля **title** предусмотреть проверку на непустое значение и ограничение до 30 символов с выводом предупреждающего хинта при нарушении
* Для поля **id** нового поста предусмотреть автоматический расчет максимального значения в соответствии с ролью и **userId**
* Для роли **admin** предусмотреть выбор **userId** из списка имеющихся в массиве данных
* Если данные изменились и валидация прошла кнопка **ОК** становится активной
* При нажатии на кнопку **ОК** происходит изменение/добавление нового поста в массив сохраненных данных. Вывод сообщения ***Пост добавлен*** или ***Пост изменен*** в зависимости от действия. Сообщение скрывается по таймауту в 10 сек и происходит переход на представление **Детализация** с фиксацией на измененном или добавленном посте
* При нажатии на кнопку **Отменить** происходит переход на представление **Детализация** с фиксацией на посте c с которого произошел переход

#### Общие требования
* Адаптивная верстка
* Удаление неиспользуемых ролью элементов
